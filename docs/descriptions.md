# Перечень тестов

## Модуль класса Board

### 1. Метод void dfs(int w, int h)

#### Тест №1.1 InitialCall (позитивный)
* _Цель_: Проверка начального вызова функции dfs.
* _Входные данные_: Начальные координаты (w, h) внутри границ доски.
* _Ожидаемый результат_: Функция успешно запускается, посещает клетки, разрушая соответствующие стенки, и завершает работу без ошибок.
* _Описание процесса_: Стандартный вызов функции с начальными координатами, проверка состояния доски после завершения работы функции.

#### Тест №1.2 BoundaryConditions (позитивный)
* _Цель_: Проверка работы функции dfs на граничных условиях.
* _Входные данные_: Начальные координаты (0, 0), (0, maxH), (maxW, 0), (maxW, maxH).
* _Ожидаемый результат_: Функция корректно обрабатывает граничные условия, разрушая стенки и посещая клетки, не выходя за пределы доски.
* _Описание процесса_: Вызов функции с граничными координатами, проверка корректного обновления состояния доски.

#### Тест №1.3 CycleDetection (негативный)
* _Цель_: Проверка на наличие циклов в графе.
* _Входные данные_: Произвольные начальные координаты (w, h) не выходящие за пределы поля.
* _Ожидаемый результат_: В графе отсутствуют циклы, каждая клетка соединена только с соседними клетками.
* _Описание процесса_: Вызов функции с произвольными координатами, анализ полученного графа на наличие циклов.

#### Тест №1.4 Randomness (позитивный)
* _Цель_: Проверка случайности выбора направления движения.
* _Входные данные_: Начальные координаты (w, h) и несколько запусков алгоритма.
* _Ожидаемый результат_: Разные запуски приводят к разным путям обхода клеток, сохраняется корректность графа.
* _Описание процесса_: Многократный запуск функции с одинаковыми начальными координатами, анализ различий в результатах.

#### Тест №1.5 InvalidStart (негативный)
* _Цель_: Проверка обработки недопустимых начальных координат.
* _Входные данные_: Начальные координаты вне границ доски (например, -1, -1 или maxW+1, maxH+1).
* _Ожидаемый результат_: Функция корректно обрабатывает недопустимые координаты, не внося изменений в состояние доски.
* _Описание процесса_: Вызов функции с недопустимыми координатами, проверка состояния доски на отсутствие изменений.

#### Тест №1.6 FullBoardVisit (позитивный)
* _Цель_: Проверка полного обхода доски.
* _Входные данные_: Начальные координаты (w, h) внутри границ доски.
* _Ожидаемый результат_: Алгоритм посещает все клетки доски, разрушаются только необходимые стенки.
* _Описание процесса_: Вызов функции с произвольными координатами, проверка посещения всех клеток.

#### Тест №1.7 PerformanceTest (позитивный)
* _Цель_: Оценка производительности функции.
* _Входные данные_: Начальные координаты (w, h) и размер доски.
* _Ожидаемый результат_: Функция выполняется за приемлемое время, даже для больших досок.
* _Описание процесса_: Вызов функции на досках разного размера, замер времени выполнения.

# Перечень тестов

## Модуль примера (Gun)

### 1. Метод `void Gun::shoot(__int64_t cur_time, bool is_running);`

#### Тест №1.1 `shoot_with_bullets_and_idle` (позитивный)
* _Цель_: Проверка стрельбы при наличии пуль и неподвижном состоянии.
* _Входные данные_: `cur_time` = 1000, `is_running` = false, `bullets_in_gun` = 5, `last_shoot_time` = 0, `bullet_duration` = 100.
* _Ожидаемый результат_: Одна пуля выпущена, `bullets_in_gun` уменьшилось на 1, `last_shoot_time` обновлено до 1000.
* _Описание процесса_: Инициализация оружия с 5 пулями, вызов метода `shoot` и проверка обновленных значений.

#### Тест №1.2 `shoot_with_bullets_and_running` (позитивный)
* _Цель_: Проверка стрельбы при наличии пуль и движении.
* _Входные данные_: `cur_time` = 1000, `is_running` = true, `bullets_in_gun` = 5, `last_shoot_time` = 0, `bullet_duration` = 100.
* _Ожидаемый результат_: Одна пуля выпущена, `bullets_in_gun` уменьшилось на 1, `last_shoot_time` обновлено до 1000, `new_dx` и `new_dy` рассчитаны с учетом разброса.
* _Описание процесса_: Инициализация оружия с 5 пулями, вызов метода `shoot` и проверка обновленных значений и направлений пуль.

#### Тест №1.3 `shoot_without_bullets` (негативный)
* _Цель_: Проверка попытки стрельбы при отсутствии пуль.
* _Входные данные_: `cur_time` = 1000, `is_running` = false, `bullets_in_gun` = 0, `last_shoot_time` = 0, `bullet_duration` = 100.
* _Ожидаемый результат_: Пуля не выпущена, `bullets_in_gun` остается 0, `last_shoot_time` не меняется.
* _Описание процесса_: Инициализация оружия с 0 пулями, вызов метода `shoot` и проверка неизмененных значений.

#### Тест №1.4 `shoot_with_recharge` (позитивный)
* _Цель_: Проверка автоматической перезарядки при отсутствии пуль и достижении времени перезарядки.
* _Входные данные_: `cur_time` = 2000, `is_running` = false, `bullets_in_gun` = 0, `last_shoot_time` = 0, `recharge_time` = 1000, `magazine_cnt` = 1.
* _Ожидаемый результат_: Магазин перезаряжен, `bullets_in_gun` обновлено до `magazine_size`, `magazine_cnt` уменьшено на 1.
* _Описание процесса_: Инициализация оружия с 0 пулями и 1 оставшимся магазином, вызов метода `shoot` и проверка обновленных значений.

#### Тест №1.5 `shoot_with_no_recharge_left` (негативный)
* _Цель_: Проверка попытки стрельбы при отсутствии пуль и отсутствии оставшихся магазинов.
* _Входные данные_: `cur_time` = 2000, `is_running` = false, `bullets_in_gun` = 0, `last_shoot_time` = 0, `recharge_time` = 1000, `magazine_cnt` = 0.
* _Ожидаемый результат_: Пуля не выпущена, `bullets_in_gun` остается 0, `magazine_cnt` остается 0.
* _Описание процесса_: Инициализация оружия с 0 пулями и 0 оставшимися магазинами, вызов метода `shoot` и проверка неизмененных значений.

#### Тест №1.6 `shoot_before_bullet_duration` (негативный)
* _Цель_: Проверка попытки стрельбы до истечения времени между выстрелами.
* _Входные данные_: `cur_time` = 50, `is_running` = false, `bullets_in_gun` = 5, `last_shoot_time` = 0, `bullet_duration` = 100.
* _Ожидаемый результат_: Пуля не выпущена, `bullets_in_gun` остается 5, `last_shoot_time` остается 0.
* _Описание процесса_: Инициализация оружия с 5 пулями и вызов метода `shoot` до истечения времени между выстрелами, проверка неизмененных значений.

#### Тест №1.7 `shoot_with_edge_case_times` (позитивный)
* _Цель_: Проверка стрельбы на граничных значениях времени.
* _Входные данные_: `cur_time` = 2147483647, `is_running` = false, `bullets_in_gun` = 5, `last_shoot_time` = 2147483547, `bullet_duration` = 100.
* _Ожидаемый результат_: Пуля выпущена, `bullets_in_gun` уменьшилось на 1, `last_shoot_time` обновлено до 2147483647.
* _Описание процесса_: Инициализация оружия с 5 пулями, вызов метода `shoot` на максимальных значениях времени и проверка обновленных значений.
